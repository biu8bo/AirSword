# 阶段 3：手势追踪 + 鼠标仿真

> 本阶段在 21 关键点基础上实现手势状态机、平滑追踪、死区、鼠标仿真，让"食指指向"真正能当鼠标用。

## 阶段目标

实现 `LingXuZhi.Core` 的手势识别、平滑追踪、坐标映射，以及 `LingXuZhi.Platform.Mouse` 的鼠标控制，完成端到端"食指当鼠标"功能。

## 前置约束（必须遵守）

1. **核心逻辑在 Core**：手势状态机、平滑滤波、死区、坐标映射全部在 `LingXuZhi.Core`，零平台依赖。鼠标控制在 `Platform.Mouse`，Windows SendInput 实现。
2. **接口解耦**：`IGestureRecognizer`（输入 21 关键点 → 输出手势状态）定义在 Core；`IMouseController`（移动/左键/右键/滚轮）定义在 Platform.Mouse。
3. **平滑优先**：鼠标移动必须平滑缓慢，禁止抖动。默认启用 EMA 滤波（系数可配），预留卡尔曼接口。
4. **死区强制**：手指在屏幕正中央死区（半径可配）内停止时，不产生移动事件。
5. **本阶段不做**：多手支持、惯用手切换、自定义手势、MediaPipe 切换（接口已预留即可）。

## 手势定义

基于 21 关键点判定（关键点编号见阶段 2 骨架拓扑）。**以下为手势优化后的现行定义**（原始定义见文末补充修改记录）：

| 手势 | 判定条件 | 触发动作 |
|------|----------|----------|
| **食指指向（移动）** | 食指(8)伸直、中指(12)收起，**其他手指不参与判定**；四指全伸直会优先判定为滚轮。指针采样点为食指指尖(8) | 移动鼠标指针 |
| **拇指-食指捏合（左键/拖拽）** | 拇指尖(4)与食指尖(8)距离 < 阈值，**其他手指不参与判定** | 原地捏合松开 → **立即**左键单击；捏住移动 → 按住左键拖拽。连续两次单击即可双击(无专用手势) |
| **比耶并拢（右键）** | 食指(8)+中指(12)伸直后指尖并拢（食中距 < 阈值），且非四指全开 | 鼠标右键点击 |
| **五指张开（滚轮向上）** | 食/中/无名/小指伸直张开，**拇指也张开** | 持续向上滚动 |
| **四指张开（滚轮向下）** | 食/中/无名/小指伸直张开，**拇指弯向掌心** | 持续向下滚动 |
| **空闲** | 以上均不满足 | 不动作 |

左右键互不干扰：左键只看拇食距离；右键看食中是否伸直并拢。

### 手指伸直/弯曲判定（建议）

- 伸直：指尖到掌根(0)的距离 > 该指根关节到掌根距离的 1.5 倍（或指尖 y 坐标超过指根关节，考虑手部朝向）。
- 弯曲：指尖在指根关节之下（朝掌心方向）。
- 具体阈值通过配置可调，先实现一套合理默认值。

### 捏合距离阈值

- 归一化坐标下的欧氏距离（相对手掌宽度,腕→中指根），默认阈值 **0.18**（偏松,提高单击命中率），可配。
- 注意：指尖关键点位于指腹中心，两指实际捏住时关键点距离约为掌宽的 10%~20%，阈值过小（如 0.05）会导致捏合永远无法触发。调试面板实时显示「捏合距离」可用于校准。

### 滚轮方向判定

- 方向由**拇指状态**决定：四指张开 + 拇指张开 → 向上；四指张开 + 拇指弯向掌心 → 向下。
- 保持手势即**持续滚动**，按固定间隔（160ms）重复触发，每次滚动行数可配（默认 3 行）。

### 左键单击 / 拖拽（拇指-食指捏合）

- 捏合确认后进入**待定态**：先不发出任何按键事件。
- **原地松开**（位移 < 拖拽阈值）→ **立即**发出左键单击（无双击窗口延迟）。
- **捏住移动**（位移 > 拖拽阈值,默认 0.05 归一化距离）→ 发出 `LeftDown` 进入拖拽态，指针跟随移动；松开捏合发出 `LeftUp` 结束拖拽。
- 需要双击时：连续做两次单击即可，由操作系统识别为双击。
- 手部丢失时状态机复位并**补发收尾事件**（拖拽中断补 `LeftUp`，捏合待定补 `LeftClick`），避免按键悬挂。

### 右键（比耶并拢）

- 食指与中指伸直（比耶）后把两指尖并拢 → 进入右键触发态，执行一次点击；保持并拢不重复触发。
- 四指全开时优先滚轮，不抢右键。

## 状态机

```
[空闲] ──食指指向──> [移动中]
   │               │
   │               ├──捏合(拇+食)──> [捏合待定] ──原地松开──> 立即单击 ──> [移动中]
   │               │                     │
   │               │                     └──捏住移动──> [拖拽中] ──松开(LeftUp)──> [移动中]
   │               │
   │               └──比耶并拢(食+中)──> [右键触发] ──释放──> [移动中]
   │
   └──手掌张开──> [滚轮模式] (五指张开持续上滚 / 拇指内收持续下滚) ──其他手势──> [空闲/移动中]
```

- 状态切换需**去抖**（连续 N 帧满足条件才切换），避免抖动误触。N 可配(默认 2)；退出捏合/拖拽用更短去抖。
- 右键点击：检测到比耶并拢进入触发态 → 执行一次点击 → 保持并拢不重复触发，释放后回到移动/空闲。
- 滚轮：进入滚轮模式后按固定间隔持续触发，切换拇指开合可直接改变滚动方向。

## 平滑追踪

| 项 | 方案 |
|----|------|
| 滤波 | EMA（指数移动平均），`out = α * raw + (1-α) * prev`，α 默认 0.3（越小越平滑越慢） |
| 预留 | 卡尔曼滤波接口 `ISmoother`，EMA 是默认实现 |
| 增益 | 鼠标位移 = 平滑后位移 × 灵敏度（可配） |
| 死区 | 平滑后位移在死区半径内 → 输出 0（不移动） |
| 中心锚定 | "屏幕正中央停止不做移动"：以画面中心为锚点，手在死区内视为静止 |

> 关键点选择：指向移动时用**食指指尖(8)**作为指针位置。

## 坐标映射

- 摄像头归一化坐标 (0~1) → 屏幕像素坐标。
- 镜像翻转：默认开启（前置摄像头自然镜像），可配。
- 映射可加非线性曲线（边缘加速/减速），本阶段先用线性，预留接口。
- 屏幕分辨率通过 `SystemParameters` / Win32 API 获取。

## 鼠标控制（Platform.Mouse）

- `IMouseController`：`MoveTo(x, y)`、`MoveBy(dx, dy)`、`LeftDown()`、`LeftUp()`、`LeftClick()`、`DoubleClick()`、`RightClick()`、`Scroll(delta)`。
- 实现 `WindowsSendInputMouseController`：用 `SendInput` (user32.dll) P/Invoke。
- 移动用绝对坐标 `MOUSEEVENTF_ABSOLUTE | MOUSEEVENTF_MOVE`（0~65535 归一化到屏幕）。
- 点击：`MOUSEEVENTF_LEFTDOWN/UP`、`MOUSEEVENTF_RIGHTDOWN/UP`。
- 滚轮：`MOUSEEVENTF_WHEEL`，delta 为 `WHEEL_DELTA` 的倍数。

## 第三阶段补充修改：手势优化（实施后追加）

初版实现完成后进行了一轮手势优化，以下为与初版规格的差异，现行行为以本节为准：

1. **左键捏合放宽**：由「食指尖(8)+拇指尖(4) 捏合且中指伸直」放宽为**只看拇指尖与食指尖距离**，其他手指不参与判定。
2. **右键改为拇指-中指捏合**：由「食指尖(8)+中指尖(12) 靠拢」改为**拇指尖(4)+中指尖(12) 捏合**，其他手指不参与判定。两种捏合同时满足阈值时取距离更近者。
3. **滚轮改为拇指定向持续滚动**：废弃「张开后上下摆动 + 回中」方案。四指张开时由拇指决定方向——拇指张开（五指张开）持续向上滚，拇指弯向掌心（四指张开）持续向下滚,固定间隔 160ms 重复触发。
4. **新增捏合拖拽**:拇指-食指捏住并移动超过拖拽阈值（默认 0.03 归一化距离，可配）→ `LeftDown` 进入拖拽,指针跟随,松开 → `LeftUp`。手部丢失时自动补发收尾事件。
5. **新增双击**：拇指-食指快速两次原地捏合（间隔在双击窗口内，默认 350ms，可配）→ 发出双击。为此单击采用**延迟确认**：原地松开后等待一个双击窗口再发出单击；待定期间指针冻结,保证落点不漂移。
6. **状态机重构**：`LeftClick` 状态拆分为 `PinchPending`（捏合待定）与 `Dragging`（拖拽中）;`MouseActionKind` 新增 `LeftDown` / `LeftUp` / `DoubleClick`。
7. **`IMouseController` 扩展**：新增 `LeftDown()`、`LeftUp()`、`DoubleClick()`。
8. **配置项新增**：`DragThreshold`（拖拽启动阈值，默认 0.03）、`DoublePinchWindowMs`（双击窗口，默认 350ms），均接入设置面板。
9. **指向判定放宽**（已被第 12 条取代）：由「食中伸直并拢 + 无名/小指弯曲 + 拇指收拢」放宽为食指、中指伸直即可,其他手指不参与判定；张开手掌（四指全伸直）优先判定为滚轮,故识别顺序调整为 捏合 > 张开 > 指向。
10. **修复镜像双重翻转**：镜像开关已在摄像头采集线程对帧做水平翻转（预览与识别共用镜像帧），坐标映射器不能按同一开关再翻转一次,否则鼠标左右反向。修正为映射器只在**镜像关闭**时补翻转，保证任何开关状态下"手向右、光标向右"。
11. **修复跟丢/坏帧导致的指针闪跳**：管线增加两道防护——
    - **跟丢宽限**：手部短暂丢失（≤6 帧）时冻结指针与状态机,不复位平滑历史；超过宽限才复位并补发收尾动作（拖拽中断 `LeftUp`、待定单击 `LeftClick`）。
    - **跳变确认**：指针单帧位移超过 0.12（归一化）视为疑似坏帧或重捕获跳变,先按上一位置输出;连续 3 帧维持在新位置才接受为真实大位移（此时复位平滑器快速跟上）。单帧坏点与"跳过去又跳回来"的闪动被完全抑制。
12. **移动手势改为食指单指,弃用「剑指」命名**：移动手势判定改为**食指伸直且中指收起**（其他手指不参与判定）,指针采样点由食中指尖中点改为**食指指尖(8)**;`FingerGeometry.SwordPointer` 更名为 `PointerPosition`,代码与 UI 中「剑指」文案统一改为「食指/指向」。中指也伸直（比耶）不再触发移动。
13. **右键改为三指捏合**：由「拇指-中指捏合」改为**拇指、食指、中指三指捏合**（拇食、拇中距离均 < 阈值）,仅拇中捏合不再触发任何手势。状态机在捏合待定态检测到转入右键时,丢弃待定单击,避免三指捏合成形过程(食指先到、中指后到)误发左键。
14. **修复捏合无法触发:默认阈值 0.05 → 0.12**：指尖关键点在指腹中心,两指实际捏住时关键点距离约为掌宽的 10%~20%,0.05 几乎永远达不到,导致单击/拖拽全部失效。默认阈值调为 0.12,设置面板滑块上限扩到 0.3;调试面板新增「捏合距离」实时显示(拇食距 · 拇中距 / 当前阈值),便于按自己手型校准。
15. **新增捏合锁定,修复点击落点漂移**：从移动切换到捏合手势时,食指弯向拇指会把指针(食指尖)带偏,导致点不到瞄准的位置。管线在平滑之前增加「捏合锁定」——拇指与食/中指尖接近到锁定区（阈值 + 0.12）时,指针锚定到**弯指前(回退 6 帧)的瞄准位置**;锁定期间指针只随**食指根(5,掌指关节)的位移**平移——弯手指不动、移手才动,因此拖拽跟手不受影响。指尖远离且状态机退出捏合相关状态后解除锁定,恢复食指尖驱动。管线顺序变为:识别 → 捏合锁定 → 跳变防护 → 平滑 → 死区 → 状态机。
16. **修复四指被识别成五指(拇指张开误判)**：拇指张开原先沿用「指尖到腕距离 > 关节2到腕距离 × 1.5」的通用伸直判定,但拇指折向掌心时指尖会横穿手掌落到无名/小指根附近,离手腕依然很远,于是被误判为张开(五指),向下滚变成向上滚。改为 `FingerGeometry.IsThumbOpen`:以**小指根(17)**为参照,拇指尖到小指根距离 > 关节2到小指根距离 × 1.1 才算张开——折向掌心时拇指尖贴近小指侧,该比值远小于 1,且参照点随手旋转,手斜着也稳定。
17. **暂时屏蔽阈值调整面板**：设置面板隐藏捏合阈值、拖拽启动阈值、双击窗口、去抖帧数及「识别」区（检测置信度 / NMS）滑块,统一使用内部默认值;保留摄像头、镜像、灵敏度、平滑、死区、滚轮行数、鼠标仿真与绘制开关。配置项本身保留,后续可随时恢复。
18. **去除专用双击手势**：删除双击窗口延迟与 `DoubleClick` 动作路径。拇食捏合原地松开后**立即**发出左键单击；需要双击时连续做两次单击即可(由 OS 识别)。退出捏合/拖拽改用更短去抖(默认去抖减 1)。默认：捏合阈值 0.18、拖拽阈值 0.05、去抖 2 帧，提高单击命中率。
19. **右键改为比耶并拢**：由三指捏合改为**食指+中指伸直后指尖并拢**(食中距 < 阈值且非四指全开)。比耶张开(指尖分开)为空闲；并拢触发一次右键。`PinchRightDistance` 调试字段改为食中指尖距。
